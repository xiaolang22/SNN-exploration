# 实验10实验报告：SpiKL-IP 信息论内在可塑性

## 1. 实验目的

在当前六模式 Izhikevich BNN 主实验基础上，引入两类内在可塑性机制：

- `rate-target IP`
- `SpiKL-IP-inspired IP`

比较四组条件：

1. `01_stdp_only`
2. `02_stdp_rate_target_ip`
3. `03_stdp_spikl_ip`
4. `04_spikl_ip_only`

核心问题：

- SpiKL-IP-inspired IP 是否能让发放率分布更接近指数分布
- SpiKL-IP-inspired IP 是否能降低沉默神经元比例
- `STDP + SpiKL-IP-inspired IP` 是否能在不引入爆发的前提下，维持或提升六模式分类准确率

## 2. 实验约束与实现原则

- 仅允许调节实验10中的 IP 参数，不修改从主实验复制过来的其他参数
- 每组实验保留独立目录、独立 `run.py`、独立 `src/`、独立 `output/`
- IP 逻辑单独放在 `src/ip.py`
- 主实验参数保持一致：
  - `n_total = 1000`
  - `n_rounds = 3`
  - `reps_per_mode = 20`
  - `stimulus_current_gain = 40.0`
  - 响应窗口 `5-55 ms`
  - 六模式固定顺序刺激

## 7. 调参过程分析

### 7.1 `target_mu_hz`

- `mu=2` 时，IP 倾向于把网络压回较低活动区间
- KL 指标会变得更好，但六模式可分性下降
- `mu=6` 时，IP 约束过松，分类也变差
- `mu=5` 在当前任务中最平衡

结论：

- 当前网络自然活动大约在 `1.3-1.6 Hz`
- `mu=5` 不是去强推 `10 Hz` 那样的高活跃状态，而是提供适度上拉
- 这个目标最有利于兼顾分类和分布整形

### 7.2 `tau_cal_ms`

- `tau=64` 最稳定
- `tau=80/96/128` 都没有带来更高的最终分类
- `tau` 过大时，钙浓度积分过慢，IP 更容易表现为全局慢性兴奋性上调

### 7.3 `eta_bias`

- `8e-5`、`1e-4`、`1.2e-4` 三档都试过
- 结果表明 `1e-4` 最合适
- 偏小会削弱整形能力，偏大则会更容易增加通道相关性

### 7.4 `eta_gain`

- `1e-5`、`2e-5`、`3e-5` 三档都试过
- `2e-5` 最平衡
- 偏大时，输入敏感性放大过多，最终分类下降

## 8. 对实验假设的判断

### H1

SpiKL-IP 可以降低沉默神经元比例，使发放率分布更接近指数分布。

判断：`支持`

依据：

- `03_stdp_spikl_ip` 的 `KL divergence` 低于基线
- `03_stdp_spikl_ip` 的 `silent neuron ratio` 低于基线

### H2

`STDP + SpiKL-IP` 在分类准确率和网络稳定性之间可能优于 `STDP only`。

判断：`支持，但提升幅度不大`

依据：

- `03` 最终 `raw count = 85.83%`
- `01` 最终 `raw count = 84.17%`
- 两者都没有 burst
- `03` 在分布整形指标上更好

### H3

如果 SpiKL-IP 过强，可能会削弱 STDP 对模式特异性的重构。

判断：`强支持`

依据：

- `mu=2`
- `tau=128`
- `eta_g=3e-5`

这些设置都说明，只要 IP 过强或过慢，就会出现“分布更规整但分类反而下降”的现象。

## 9. 总结

本次实验表明：

- 在当前六模式任务和 Izhikevich BNN 框架下，`SpiKL-IP-inspired IP` 是可行的
- 它并不能带来大幅度分类提升，但可以在不引入爆发的前提下：
  - 略微提升最终 `raw count` 分类准确率
  - 降低沉默神经元比例
  - 让发放率分布更接近目标指数分布

最优参数为：

```text
target_mu_hz = 5.0
tau_cal_ms = 64.0
eta_bias = 1e-4
eta_gain = 2e-5
bias_max = 2.0
gain_max = 1.3
```

因此，实验10目前可以给出的最稳妥结论是：

`STDP + SpiKL-IP-inspired IP` 在当前任务中优于 `STDP only`，但优势较弱，主要体现在“分布整形 + 小幅分类提升”，而不是显著提升分类上限。

## 10. 后续建议

- 不建议继续在单一 `seed=42` 上做更细粒度参数抠调
- 下一步最值得做的是多 `seed` 重复
- 重点比较：
  - `01_stdp_only`
  - `03_stdp_spikl_ip`

建议统计：

- 跨 seed 平均 `raw count`
- 跨 seed 平均 `KL divergence`
- 跨 seed 平均 `silent neuron ratio`
- 跨 seed 标准差

如果 `03` 在跨 seed 层面仍稳定优于 `01`，则实验10可以认为得到较完整支持。
