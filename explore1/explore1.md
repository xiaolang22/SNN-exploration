# explore1：训练期在线拓扑可塑性验证实验方案

## 1. 实验目标

本实验首先验证训练期在线拓扑可塑性本身是否有效，然后进一步与传统突触权重可塑性作直接对比。

```text
在随机递归 SNN/BNN 中，引入训练过程中在线运行的拓扑可塑性机制后，
是否能够提升六模式分类性能、改善网络动态稳定性，并重组出更有利于信息传输的功能结构？
同时，与经典 STDP 突触权重可塑性相比，拓扑可塑性是否表现出更强的性能收益？
```

本实验的定位是一个机制验证与机制对比实验，而不是完整性能上限实验。核心目的是验证“训练中根据活动模式逐步重连网络拓扑”这一路线本身是否有效，并与 STDP 的效果做最小变量比较。

---

## 2. 核心思想

### 2.1 机制出发点

实验11已经说明，在当前任务下，拓扑结构本身会显著影响分类性能。原始随机网络虽然具有一定 reservoir 计算能力，但其高价值信息通路往往被埋在大量低效或冗余连边中。

本实验希望验证如下思想：

```text
如果某些神经元之间经常通过较长的多跳路径传递有效刺激信息，
则训练过程中可以逐步新增一条更短的捷径连接；
如果某些原有连边长期不参与有效传播，则可以将其移除，
从而让网络在总连接预算近似不变的前提下，自发形成更高效的拓扑。
```

### 2.2 本实验中的保守版本

为了先验证机制是否有用，本实验不直接实现“释放整颗神经元并重新编入其他链路”的强版本，而采用更稳妥的弱版本：

```text
释放的是连接预算，而不是神经元本体。
```

即：

1. 检测训练中高频出现的有效传播对
2. 为这些传播对创建 shortcut 候选边
3. 删除长期低贡献的旧边
4. 保持总边数基本恒定

这样更容易控制变量，也更容易将性能变化归因于“拓扑重连”本身。

---

## 3. 研究问题与假设

### 3.1 研究问题

本实验重点回答以下问题：

1. 仅引入在线拓扑可塑性，是否能提高六模式分类准确率？
2. 该机制是否能提高刺激响应的可分离性与 trial-to-trial 稳定性？
3. 该机制是否会改变网络的图结构特征，如平均路径长度、聚类系数、模块化或连接距离分布？
4. 该机制带来的收益是否伴随更低的沉默神经元比例、更高的利用率，或更强的爆发倾向？
5. 与随机网络 + STDP 相比，在线拓扑可塑性是否能取得更高的最终分类准确率或更好的稳定性？

### 3.2 实验假设

假设 H1：

```text
随机网络 + 在线拓扑可塑性组的最终分类准确率将高于随机网络 + 无可塑性组。
```

假设 H2：

```text
在线拓扑可塑性组在训练过程中会逐步提高模式分离性，
其类别间距离、可解码性和刺激响应一致性将优于对照组。
```

假设 H3：

```text
在线拓扑可塑性将使网络形成更高效的连边组织，
表现为平均有效路径长度下降，部分高价值节点之间出现新的捷径连接。
```

假设 H4：

```text
若机制设计合理，网络利用率会提高，但不应显著增加全局爆发和异常同步。
```

假设 H5：

```text
随机网络 + 在线拓扑可塑性组的最终分类性能将优于随机网络 + STDP 组。
```

## 4. 实验分组

本实验设置三组。

### 4.1 A组：随机网络 + 无可塑性

```text
网络拓扑：随机初始化
训练期间：拓扑固定
权重：固定
STDP：关闭
其他塑性：关闭
```

作用：

```text
作为绝对基线，用于衡量仅靠随机 reservoir 动力学时的分类能力与响应结构。
```

### 4.2 B组：随机网络 + 在线拓扑可塑性

```text
网络拓扑：与 A 组同分布随机初始化
训练期间：允许按规则新增和删除连边
权重：固定
STDP：关闭
其他塑性：关闭
```

作用：

```text
验证在线拓扑可塑性本身是否足以带来性能提升。
```

### 4.3 C组：随机网络 + STDP

```text
网络拓扑：与 A 组同分布随机初始化
训练期间：拓扑固定
权重：允许按 STDP 规则更新
STDP：开启
其他塑性：关闭
```

作用：

```text
作为传统突触可塑性对照组，用于比较“只改权重”和“只改拓扑”哪一种更有效。
```

### 4.4 控制原则

除可塑性机制开关外，以下条件必须完全一致：

1. 网络规模
2. 神经元参数
3. 兴奋/抑制比例
4. 初始连接概率或初始边数
5. 刺激协议
6. 训练轮数
7. 响应窗口
8. 特征提取方式
9. 分类器与交叉验证方法
10. 随机种子管理方式

## 5. 机制定义

### 5.1 总体原则

本机制是一个训练期间周期性触发的结构重连模块，目标不是全局搜索最优图，而是根据近期活动统计做局部、渐进式重连。

需要满足四个原则：

1. 在线运行，不依赖离线进化
2. 局部更新，避免每次大规模改图
3. 总连接预算近似恒定
4. 重连速度慢于单次刺激响应速度

### 5.2 “有效传播对”的定义

为了避免直接在所有长路径上做组合爆炸搜索，本实验先采用简化定义：

对于任意两个神经元 `i -> j`，若在训练阶段多次出现如下现象，则认为它们构成“有效传播对”候选：

1. 在同类刺激试次中，`i` 的放电经常先于 `j`
2. 两者的时延落在一个稳定窗口内
3. 该先后关系在多轮试次中重复出现
4. `i` 与 `j` 当前没有直接连边

这本质上是在寻找：

```text
经常通过中间节点间接耦合、且具有稳定因果时序关系的神经元对。
```

### 5.3 候选 shortcut 分数

可为每个候选对定义一个 shortcut score：

```text
S_short(i, j) =
  alpha * coincidence(i, j)
+ beta  * consistency(i, j)
+ gamma * class_specificity(i, j)
+ delta * path_gain(i, j)
- eta   * distance_cost(i, j)
```

其中：

1. `coincidence(i, j)`：训练中 `i` 领先 `j` 的共现频率
2. `consistency(i, j)`：该时延关系在 trial 间的一致性
3. `class_specificity(i, j)`：该关系是否主要出现在特定刺激模式下
4. `path_gain(i, j)`：若二者当前最短有效传播路径较长，则压缩收益更大
5. `distance_cost(i, j)`：空间上过长的新增连接需要惩罚，避免生成不合理超远程边

在第一版实现中，可以只保留前三项，简化为：

```text
S_short(i, j) = alpha * coincidence + beta * consistency + gamma * class_specificity
```

### 5.4 弱连接/低贡献边分数

为每条现有边 `e = (u, v)` 定义待删除分数：

```text
S_prune(u, v) =
  a * low_usage(u, v)
+ b * low_causality(u, v)
+ c * redundant_edge(u, v)
+ d * silent_association(u, v)
```

含义：

1. `low_usage`：该边对应的前后神经元很少出现有效时序传播
2. `low_causality`：边两端缺乏稳定时间先后关系
3. `redundant_edge`：该边功能可被其他并行路径替代
4. `silent_association`：该边长期连接沉默或近沉默节点

### 5.5 结构更新规则

每经过固定数量 trial 或每完成一轮训练后，执行一次结构更新：

1. 统计最近一个窗口内的神经活动
2. 计算所有候选 shortcut 分数
3. 选出前 `K_add` 条未连接候选边
4. 计算现有边的删除分数
5. 选出前 `K_prune` 条待删除边
6. 检查拓扑约束后，执行新增与删除

默认采用：

```text
K_add = K_prune
```

从而保持总边数不变。

### 5.6 约束条件

为避免结构更新破坏网络基本动力学，需要加入硬约束：

1. 不允许自连接
2. 不允许重复边
3. 保持兴奋/抑制类型约束不变
4. 每次更新单元内，单个神经元入度/出度变化不超过阈值
5. 不允许使任一神经元完全失去全部输入或全部输出
6. 新增超远程连接时加入惩罚或上限
7. 每轮只更新总边数的一小部分，如 `1% - 5%`

### 5.7 慢时标更新

拓扑可塑性不能每个 spike 都更新，而应处于比神经动力学更慢的时标。建议：

1. 每 `N` 个 trial 更新一次
2. 或每一轮训练结束后更新一次

推荐先使用更稳的方案：

```text
每轮训练结束后更新一次拓扑
```

因为这样最容易分析“当前一轮活动统计导致下一轮结构变化”的因果关系。

## 6. 与现有代码框架的关系

### 6.1 建议复用的基线

优先复用主实验或实验11中的如下部分：

1. 网络构建流程
2. 六模式刺激生成流程
3. trial 记录与响应特征提取流程
4. 分类与绘图流程
5. 输出结果组织方式

这样可以保证 explore1 和已有实验具有直接可比性。

### 6.2 本实验应避免复用的内容

以下机制在 explore1 中应明确关闭，或仅在专门对照组中单独开启：

1. STDP 以外的权重学习机制
2. SpiKL-IP
3. 结构进化搜索
4. degree-matched random 特殊对照
5. 任何会改变权重的额外学习机制

否则就无法把提升归因于在线拓扑可塑性或 STDP 本身。

## 7. 具体实现方案

### 7.1 目录建议

建议建立如下目录：

```text
explore1/
  explore1.md
  src/
    config.py
    topology_plasticity.py
    metrics_topology.py
    run.py
  output/
    ...
```

如果你希望与 `experiment11` 高度复用，也可以直接复制一个最简实验模板，并仅替换结构模块。

### 7.2 `config.py` 建议新增项

建议新增：

```python
topology_plasticity.enabled = True / False
topology_plasticity.update_interval_rounds = 1
topology_plasticity.max_add_per_update = 20
topology_plasticity.max_prune_per_update = 20
topology_plasticity.max_edge_fraction_per_update = 0.02
topology_plasticity.min_causal_count = 3
topology_plasticity.causal_delay_ms = (1, 20)
topology_plasticity.use_class_specificity = True
topology_plasticity.distance_penalty = 0.1
topology_plasticity.protect_input_output_degree = 1
topology_plasticity.keep_total_edges_constant = True
```

第一版不要把参数做得太多，避免调参空间失控。

### 7.3 `topology_plasticity.py` 模块职责

该模块建议包含以下函数：

1. `collect_activity_statistics(...)`
2. `score_shortcut_candidates(...)`
3. `score_existing_edges_for_pruning(...)`
4. `select_rewiring_operations(...)`
5. `apply_rewiring(...)`
6. `summarize_topology_change(...)`

其中：

#### `collect_activity_statistics(...)`

输入：

1. 当前轮所有 trial 的 spike 记录
2. 网络当前连边
3. trial 标签

输出：

1. 神经元对的时序共现统计
2. 每类刺激下的局部因果关系统计
3. 边使用率统计
4. 沉默节点相关统计

#### `score_shortcut_candidates(...)`

输出所有未连接神经元对的候选分数表。

第一版可限制候选空间：

1. 只考虑空间距离不超过阈值的神经元对
2. 只考虑在活动统计中出现过明显先后关系的神经元对

这样可以显著降低复杂度。

#### `score_existing_edges_for_pruning(...)`

输出所有现有边的删除优先级分数。

#### `select_rewiring_operations(...)`

执行：

1. 候选新增边排序
2. 候选删除边排序
3. 约束过滤
4. 一一配对生成重连操作

#### `apply_rewiring(...)`

真正修改 adjacency / connection list。

### 7.4 `run.py` 的执行流程

建议流程如下：

1. 初始化随机网络
2. 对 A、B 两组使用同一随机种子生成初始拓扑
3. 运行预热阶段
4. 进入正式训练轮次
5. 每轮收集所有 trial 响应
6. 提取特征并做轮次分类评估
7. 若为 B 组，则在轮末执行一次拓扑更新
8. 保存每轮拓扑快照与统计指标
9. 训练结束后做最终测试与汇总

关键点：

```text
A、B 两组应尽量共享相同的初始网络实例，
唯一差别是 B 组在每轮后允许更新拓扑。
```

这样最有利于控制变量。

## 8. 训练与评估流程

### 8.1 建议实验阶段

#### 阶段1：预热

作用：

```text
让随机网络进入稳定活动区间，避免初始瞬态直接影响拓扑统计。
```

建议：

1. 与主实验一致的预热流程
2. 预热阶段不执行拓扑更新

#### 阶段2：训练

作用：

```text
在重复刺激过程中积累活动统计，并逐轮更新拓扑。
```

建议：

1. 使用与主实验一致的 `n_rounds`
2. 每轮结束后更新一次拓扑

#### 阶段3：冻结测试

作用：

```text
在最终拓扑固定的条件下评估分类性能与动态稳定性。
```

建议：

1. B 组在最后一轮拓扑更新后冻结结构
2. 使用统一测试集或统一 trial 汇总流程评估

### 8.2 建议训练轮数

为了便于先观察趋势，可先用：

```text
n_rounds = 3
```

如果能看到趋势，再扩展到 `5` 或 `8` 轮分析“拓扑是否持续改善或后期过拟合”。

### 8.3 多随机种子重复

单次随机网络结果波动可能很大，因此必须做多 seed 重复。

建议：

```text
n_seeds = 10
```

每个 seed 下：

1. 先生成一个随机初始拓扑
2. 复制给 A、B 两组
3. 分别运行

最终比较两组配对差异。

## 9. 评价指标

### 9.1 主指标

主指标只保留最核心的分类性能指标：

1. `raw spike count` 的 10-fold CV 准确率
2. `V100` 的 10-fold CV 准确率

如果你主实验已有固定主指标，explore1 最好完全复用。

### 9.2 过程指标

用于回答“为什么会提升”：

1. 每轮分类准确率曲线
2. 每轮类别分离度
3. 每轮 trial-to-trial 一致性
4. 每轮沉默神经元比例
5. 每轮 burst ratio 或 burst-like trial ratio
6. 每轮平均发放率

### 9.3 拓扑指标

必须记录的结构指标：

1. 总边数
2. 平均入度/出度
3. 度分布标准差
4. 平均连接距离
5. 聚类系数
6. 平均最短路径长度或近似有效路径长度
7. 小世界性指标
8. 模块化指标
9. 新增边数量与删除边数量
10. 每轮保留下来的 shortcut 边比例

### 9.4 机制特异指标

为了验证你的想法本身，建议增加以下专门指标：

1. 新增 shortcut 边在更新前是否确实对应较强时序共现
2. 新增 shortcut 边是否在后续轮次中持续被使用
3. 被删除边是否确实具有低贡献特征
4. 重连前后，高价值节点对的有效路径长度是否下降

这些指标直接决定你能不能说“机制按预期工作了”，而不只是“碰巧准确率涨了”。

## 10. 统计分析

### 10.1 基本比较方式

由于 A、B 两组共享同一初始 seed，建议用配对统计。

比较：

1. 每个 seed 的最终准确率差值
2. 每个 seed 的最终分离度差值
3. 每个 seed 的最终 burst ratio 差值
4. 每个 seed 的拓扑指标变化量差值

### 10.2 建议统计方法

可使用：

1. 配对 t 检验
2. 或 Wilcoxon signed-rank test

同时报告：

1. 均值
2. 标准差
3. 效应量
4. 95% 置信区间

### 10.3 关键判定标准

若满足以下条件，可认为该机制初步有效：

1. B 组最终准确率显著高于 A 组
2. B 组分离度提高
3. B 组未出现明显更强的爆发或塌缩
4. B 组拓扑指标显示出有方向性的结构重组

## 12. 可能结果及解释

### 12.1 若 B 组显著优于 A 组

说明：

```text
即使不依赖权重可塑性，仅靠训练中在线拓扑重连，
随机递归 SNN 也能逐步形成更适合当前任务的信息传播结构。
```

这将直接支持你的基本想法。

### 12.2 若 B 组准确率不变，但分离度和稳定性提高

说明：

```text
该机制可能先改善网络动态组织，但当前特征提取或分类器尚未完全利用这些变化。
```

此时机制仍可能是有效的，只是输出读出层没有充分体现。

### 12.3 若 B 组准确率提高，但 burst 同步明显升高

说明：

```text
捷径生成确实增强了传播效率，但也可能过度提高了局部耦合强度，
导致网络更易同步化。
```

后续需要加入更强的 burst 抑制项或结构更新约束。

### 12.4 若 B 组不优于 A 组

可能原因：

1. 候选 shortcut 规则噪声太大
2. 更新过快，破坏了 reservoir 动力学
3. 剪枝规则误删了有用连接
4. 当前任务本身不需要路径压缩
5. 主导性能的不是结构重连，而是权重微调

这并不直接否定你的想法，但说明第一版规则还不够准确。

## 13. 风险与对策

### 13.1 风险：候选空间过大

问题：

```text
所有未连接神经元对都打分会导致计算量过大。
```

对策：

1. 只统计活跃节点对
2. 只考虑空间距离低于阈值的节点对
3. 只保留高共现候选对

### 13.2 风险：更新后网络塌缩

问题：

```text
过快重连可能让网络突然进入沉默或爆发状态。
```

对策：

1. 每轮只更新 1% 到 2% 的边
2. 加入最小入度/出度保护
3. 一旦发放率异常，回滚本轮更新

### 13.3 风险：性能提升只是偶然

问题：

```text
单次 seed 可能刚好更幸运。
```

对策：

1. 做多 seed 重复
2. 使用配对统计
3. 保存每轮结构变化日志

### 13.4 风险：机制解释力不足

问题：

```text
就算准确率涨了，也可能无法证明是“捷径形成”在起作用。
```

对策：

1. 必须记录新增边的因果分数
2. 必须分析有效路径长度变化
3. 必须展示代表性 seed 的结构演化案例

## 14. 最小可行实现版本

为了尽快得到第一批结果，建议先做一个最小版本：

### 14.1 最小版本规则

1. 每轮结束后更新一次拓扑
2. 根据神经元对的时序共现频率给未连接对打分
3. 新增前 `K` 条最高分边
4. 删除前 `K` 条最低使用率边
5. 保持总边数不变
6. 不做复杂路径搜索
7. 不做神经元释放，只做连边重分配

### 14.2 最小版本意义

如果这个简化版本就已经有效，说明：

```text
你的核心方向是对的，
后面再逐步加入“路径长度收益”“类特异性”“近似功能等效”等更强规则即可。
```

如果最小版本无效，再决定是否有必要上更复杂的路径级机制。

## 15. 后续扩展方向

如果 explore1 验证成功，后续可以自然扩展为：

1. 随机网络 + STDP

```text
拓扑可塑性是单独有效，还是主要通过与突触可塑性协同发挥作用？
```

但这一步不属于 explore1。

## 16. 本实验的结论目标

explore1 的成功标准不是“做到最强性能”，而是得到一个明确结论：

```text
在当前六模式 BNN/SNN 框架下，
训练期在线拓扑可塑性本身是否构成一种有效的学习机制。
```

如果答案是肯定的，那么 explore1 就为后续更复杂的“结构与权重协同可塑性”实验提供了坚实基础。
