## 7. STDP 必要性与训练后固定权重实验

### 1.1 研究问题

当前主实验显示网络可以区分六种刺激模式，但必须证明这种能力不只是来自刺激电压和空间位置本身，而是和 STDP 诱导的突触重构有关。

需要回答两个问题：

1. 没有 STDP 时，六模式分类表现如何？
2. 经过 STDP 训练后的网络，在关闭 STDP、固定权重后，是否仍然保留已学习到的模式可分性？

### 1.2 实验假设

假设 H1：

```text
启用 STDP 的网络在训练后表现出更高的模式分类准确率。
```

假设 H2：

```text
训练 3 轮后固定权重并关闭 STDP，网络仍能在随机刺激测试中保持较高分类准确率，说明学习结果已经写入突触权重，而不是只依赖训练时瞬时动态。
```

### 1.3 实验组设计

建议设置三组。

**A. Full-STDP 主实验组**

```text
预热：STDP 开启
训练：STDP 开启
测试：使用每轮响应做分类
```

目的：作为主结果。

**B. No-STDP 对照组**

```text
预热：STDP 关闭
训练：STDP 关闭
测试：同样刺激流程，同样分类方法
```

目的：检验没有突触学习时，刺激本身能产生多强的可分响应。

实现方式：

```python
cfg.plasticity.stdp.enabled = False
```

注意：如果希望预热阶段也完全无 STDP，需要从实验开始就关闭，而不是只在训练阶段关闭。

**C. Train-then-Freeze 测试组**

流程：

```text
1. 按主实验训练 3 轮，STDP 开启。
2. 保存第 3 轮后的权重。
3. 关闭 STDP，固定权重。
4. 使用随机顺序刺激重新测试若干 trial。
5. 用测试 trial 的响应做分类。
```

目的：检验 STDP 训练后的权重是否形成稳定记忆。

建议测试集：

```text
每模式 30-50 trial
刺激顺序随机打乱
不再更新权重
```

### 1.4 关键控制

No-STDP 组和 Full-STDP 组应共享相同的：

```text
随机种子
神经元位置
连接拓扑
初始权重
刺激热点
刺激校准参数
```

否则差异可能来自网络初始化，而不是 STDP。

最好采用配对实验：

```text
同一个 seed 下运行 Full-STDP 和 No-STDP
再比较两者最终准确率
```

### 1.5 评价指标

核心指标：

```text
第 1-3 轮 raw count 准确率
第 1-3 轮 V100 准确率
最终测试集准确率
最后一轮混淆矩阵
权重分布变化
平均发放率变化
```

建议增加：

```text
ΔAccuracy = Accuracy_full - Accuracy_no_stdp
权重变化幅度：mean(|W_after - W_before|)
权重分布 KL divergence 或 Wasserstein distance
```

### 1.6 推荐图表

论文图可以设计为：

```text
图 A：Full-STDP vs No-STDP 每轮准确率曲线
图 B：Train-then-Freeze 随机测试混淆矩阵
图 C：预热后、第 3 轮后兴奋性权重分布
图 D：每个 seed 的最终准确率配对散点图
```

### 1.7 预期结果与解释

理想结果：

```text
Full-STDP > No-STDP
Train-then-Freeze 仍明显高于随机基线
```

可解释为：

```text
STDP 将重复刺激诱发的时空活动模式写入兴奋性突触结构，使网络在固定权重测试阶段仍保留模式区分能力。
```

如果 No-STDP 也很高：

```text
说明六种刺激本身已经强可分，STDP 的贡献需要通过泛化测试、扰动鲁棒性或权重冻结测试进一步证明。
```

如果 Freeze 后下降明显：

```text
说明分类可能依赖训练过程中的瞬时状态，而不是稳定权重记忆，需要重新设计测试阶段或延长训练。
```

## 8. 模式间扰动刺激仿真实验

### 2.1 实验目的

在主实验的固定六模式训练流程中，额外加入“模式间扰动刺激”，观察扰动是否能改善分类准确率、减少网络爆发、提升跨 seed 稳定性。

重点比较：

```text
无扰动训练 vs 模式间扰动训练
```

### 2.2 基本流程

主实验一轮训练顺序为：

```text
A1_100mV × 20
A1_300mV × 20
A1_500mV × 20
A2_100mV × 20
A2_300mV × 20
A2_500mV × 20
```

扰动版改为：

```text
A1_100mV × 20
Perturbation block
A1_300mV × 20
Perturbation block
A1_500mV × 20
Perturbation block
A2_100mV × 20
Perturbation block
A2_300mV × 20
Perturbation block
A2_500mV × 20
```

其中每个 `Perturbation block` 不作为分类样本，只用于改变网络状态并参与 STDP 更新。

### 2.3 扰动 block 设置

默认扰动 block：

```text
随机扰动脉冲数：N ~ UniformInteger(4, 10)
每个扰动脉冲幅度：V ~ Uniform(200, 600) mV
扰动脉冲间隔：10 ms
扰动脉冲形式：双相，+1.0 / -1.0
扰动脉冲位置：随机选择或按预设策略选择
```

仿真中继续使用当前主实验的 1 ms 正相 + 1 ms 负相双相脉冲。由于当前时间步为 `dt_ms = 1.0`，不再细分到微秒尺度。

### 2.4 扰动位置策略

建议先做 P1，后续再扩展 P2/P3。

**P1：随机区域扰动**

```text
每个扰动脉冲随机选择 A1 或 A2 作为中心。
```

**P2：全平面随机扰动**

```text
每个扰动脉冲在 3.85 × 2.10 mm 平面内随机选择一个刺激中心。
```

**P3：非输入区域扰动**

```text
扰动中心避开 A1/A2，在其他区域随机选取。
```

### 2.5 扰动强度分组

建议做四组：

```text
No perturbation：无扰动，当前主实验
Weak perturbation：4 个脉冲，200-300 mV
Medium perturbation：4-10 个脉冲，200-600 mV
Strong perturbation：10 个脉冲，500-800 mV
```

默认优先实现：

```text
No perturbation
Medium perturbation
```

### 2.6 对照组设计

至少设置三组：

```text
1. Structured only
   只有六模式结构化刺激，无扰动。即当前主实验。

2. Structured + perturbation
   模式块之间插入 4-10 个 200-600 mV 随机双相扰动脉冲。

3. Perturbation only
   不呈现六种结构化模式，只给同等数量的随机扰动刺激。
```

如果时间允许，再增加：

```text
4. Structured + fixed perturbation
   扰动脉冲数量、幅度和位置固定，不随机。

5. Structured + shuffled perturbation timing
   扰动仍存在，但不放在模式之间，而是随机插入训练过程。
```

### 2.7 训练期扰动

```text
训练阶段：结构化模式块之间插入 perturbation block
分类样本：只取结构化六模式 trial
扰动样本：不进入分类标签
STDP：结构化刺激和扰动刺激期间都开启
```

每轮仍然有 120 个结构化 trial 用于分类：

```text
6 模式 × 20 trial = 120 个分类样本
```

扰动 block 只影响训练过程中的网络状态，不增加分类类别。

### 2.8 可选：扰动训练后固定权重测试

```text
1. 使用 Structured + perturbation 训练 3 轮。
2. 关闭 STDP，固定权重。
3. 用随机顺序六模式刺激重新测试。
4. 比较无扰动训练和扰动训练后的固定权重测试准确率。
```

该实验用于判断扰动训练是否形成更稳定的权重记忆。

### 2.9 评价指标

分类指标：

```text
每轮 raw count 10-fold CV 准确率
每轮 V100 10-fold CV 准确率
out-of-fold 混淆矩阵
最终轮准确率
跨 seed 平均准确率和标准差
```

网络稳定性指标：

```text
平均发放率 MFR
burst ratio：高发放 trial 占比
最大 trial MFR
通道活动热图是否出现全局爆发
```

功能连接指标：

```text
平均相关 avg_correlation
全局效率 global_efficiency
模块化 modularity
```

扰动效果判断：

```text
扰动组准确率是否高于无扰动组
扰动组跨 seed 方差是否更小
扰动组 channel heatmap 是否更少出现全局爆发
扰动组功能连接指标是否更稳定
```

建议定义 burst-like trial：

```text
若某 trial 在 5-55 ms 内记录通道平均发放率 > 100 Hz，则记为 burst-like trial。
```

也可以根据实际主实验分布调整阈值。

### 2.10 推荐图表

建议输出：

```text
图 A：Structured only vs Structured + perturbation 的训练流程示意
图 B：两组每轮分类准确率曲线
图 C：两组 channel heatmap 对比
图 D：两组功能连接指标随轮次变化
图 E：两组 burst ratio 对比
```

如果做多强度扰动，增加：

```text
不同扰动强度下的最终准确率
不同扰动位置策略 P1/P2/P3 的比较
Perturbation only 负对照结果
out-of-fold 混淆矩阵
```

### 2.11 预期结果

理想结果：

```text
Structured + perturbation 的最终准确率高于 Structured only
扰动组的跨 seed 方差更小。
扰动组 channel heatmap 更少出现全局同步爆发。
扰动组 global_efficiency 上升，modularity 下降或保持较低。
Perturbation only 接近随机基线或明显低于结构化组。
```

如果扰动组准确率不升高，但稳定性变好，也有价值：

```text
扰动刺激没有显著提高分类上限，但降低了 burst-like trial 比例并减少了跨 seed 方差，说明扰动主要发挥稳定化和去同步作用，而非直接增加分类信息。
```

如果强扰动使准确率下降：

```text
说明扰动存在最优强度区间。过弱扰动不足以改变网络状态，过强扰动会破坏结构化刺激响应。
```

## 9. 可塑性机制组合消融实验

### 3.1 研究问题

在主实验流程不变的前提下，比较不同可塑性机制组合对六模式分类、网络稳定性和功能连接重组的影响。

重点回答：

```text
STDP 是否是分类能力提升的主要来源？
STP 是否能减少连续刺激引起的高发放和爆发？
IP 是否能稳定平均发放率并降低跨 seed 差异？
组合机制是否比单一机制更稳定？
```

### 3.2 实验假设

假设 H1：

```text
STDP 是模式学习的主要来源。
```

假设 H2：

```text
STP 和 IP 不一定直接提高分类准确率，但可能提高发放率稳定性、抑制爆发和提升跨 seed 稳定性。
```

假设 H3：

```text
STDP + STP + IP 的完整模型可能在准确率、稳定性和鲁棒性之间取得更好平衡。
```

### 3.3 实验条件

建议设置七组：

```text
1. No plasticity：无 STDP、无 STP、无 IP
2. STDP only
3. STDP + STP
4. STDP + IP
5. STDP + STP + IP
6. STP only
7. IP only
```

如果论文篇幅有限，主文放前 5 组，补充材料放 STP only 和 IP only。

### 3.4 每种机制的作用定位

**STDP**

```text
长时程改变兴奋性突触权重
预期主要影响学习和分类能力
```

**STP**

```text
短时程调节 spike 后有效突触释放量
预期抑制连续高频发放造成的兴奋性爆发
```

**IP**

```text
根据神经元活动调节内在偏置或兴奋性
预期维持目标发放率，减少跨 seed 活动差异
```

### 3.5 实验流程

每个条件都使用同样的刺激流程和评估流程：

```text
预热 20 s
选热点并校准刺激
3 轮训练，每模式每轮 20 trial
5-55 ms 响应窗口
10 折交叉验证
```

每个条件只改变可塑性机制开关，不改变刺激模式、训练轮数、响应窗口和分类器。

### 3.6 机制开关建议

机制开关可以设计为：

```text
STDP enabled: True / False
STP enabled: True / False
IP enabled: True / False
```

STDP：

```text
只更新兴奋性突触权重。
关闭时，突触权重固定不变。
```

STP：

```text
不改变长期权重，只改变 spike 传播时的短时程有效释放量。
```

IP：

```text
根据神经元近期活动调节静态偏置项，使发放率靠近目标水平。
```

对带 IP 的条件，建议采用：

```text
预热阶段：开启 IP
训练阶段：开启 IP
```

这样可以直接评估 IP 对训练全过程稳定性的贡献。如果担心 IP 过强影响 STDP 学习，可额外做一组“只在预热开启 IP”的补充实验。

### 3.7 评价指标

分类指标：

```text
每轮 raw count 10-fold CV 准确率
每轮 V100 10-fold CV 准确率
最后一轮 out-of-fold 混淆矩阵
跨 seed 平均准确率和标准差
```

稳定性指标：

```text
平均发放率均值
平均发放率变异系数
最大 trial 发放率
通道热图是否出现全局爆发
权重是否堆到 w_max
跨 seed 准确率标准差
```

功能连接指标：

```text
平均相关 avg_correlation
全局效率 global_efficiency
模块化 modularity
```

建议定义爆发判据：

```text
若某 trial 的记录通道平均发放率 > 100 Hz，则标记为 burst-like trial
burst ratio = burst-like trials / all trials
```

### 3.8 推荐图表

```text
图 A：不同机制组的最终准确率箱线图
图 B：不同机制组的发放率轨迹
图 C：不同机制组的 burst ratio
图 D：不同机制组的权重分布
图 E：不同机制组的功能连接指标
```

如果做多 seed，建议增加：

```text

图 G：不同机制组的准确率-稳定性二维图
```

### 3.9 预期结果

可能出现的结果：

```text
STDP only 准确率较高，但稳定性一般
STDP + STP 爆发更少，准确率更稳
STDP + IP 发放率更接近目标水平
STDP + STP + IP 综合表现最好
No plasticity 明显弱于 STDP 相关组
```

如果 STP 或 IP 降低准确率，但明显减少爆发或降低跨 seed 方差，也应保留结果：

```text
短时程或内在调节机制提高了网络稳定性，但可能牺牲部分模式可分性。
```

最终报告时应把“分类能力”和“网络稳定性”分开评价，不只用准确率判断机制优劣。

## 10. SpiKL-IP 信息论内在可塑性实验

### 10.1 实验目的

在当前六模式主实验基础上，引入一种以发放率分布优化为目标的 IP 机制，比较它与普通目标发放率 IP 的差异。

核心问题：

```text
普通 IP：让每个神经元的平均发放率靠近目标值
SpiKL-IP：让每个神经元的输出发放率分布逼近指数分布
```

因此该实验不只看平均发放率是否稳定，还要看神经元群体是否形成更高熵、更分散、更稳定的响应分布。

### 10.2 实验假设

假设 H1：

```text
SpiKL-IP 可以降低网络爆发和沉默神经元比例，使整体发放率分布更接近指数分布。
```

假设 H2：

```text
STDP + SpiKL-IP 在分类准确率和网络稳定性之间可能优于 STDP only。
```

假设 H3：

```text
如果 SpiKL-IP 过强，可能会削弱 STDP 对特定刺激模式的权重重构，导致分类准确率下降。
```

### 10.3 机制实现方案

SpiKL-IP 原始规则针对 LIF 神经元，通过调节每个神经元的等效漏电阻 `R` 和膜时间常数 `tau_m` 改变输入-输出函数。

当前主实验使用 Izhikevich 神经元，因此建议分两层实现。

**方案 A：Izhikevich 兼容代理版本**

优先实现这个版本，因为它可以直接接入当前主实验。

每个神经元维护一个钙浓度变量：

```text
C_cal(t + dt) = C_cal(t) - C_cal(t) / tau_cal * dt + spike(t)
y(t) = C_cal(t) / tau_cal
```

其中：

```text
C_cal：近期发放活动的连续估计
tau_cal：钙浓度衰减时间常数
y：瞬时发放率估计
```

然后用 `y` 与目标指数分布的均值 `mu` 构造 IP 调节项，更新每个神经元的内在兴奋性参数。

建议在 Izhikevich 模型中调节：

```text
bias_current_i：每个神经元的偏置电流
input_gain_i：每个神经元接收外部输入和突触输入的增益
```

更新方向：

```text
神经元长期沉默：提高 bias_current_i 或 input_gain_i
神经元长期过度发放：降低 bias_current_i 或 input_gain_i
发放率分布偏离指数目标：按 KL 损失方向调节兴奋性
```

该版本应命名为：

```text
SpiKL-IP-inspired Izhikevich IP
```

因为它保留了钙浓度发放率估计、指数目标分布和 KL 优化思想，但没有直接使用 LIF 的 `R` / `tau_m` 推导式。

**方案 B：LIF 储备池严格版本**

作为补充实验，可以新增一个 LIF reservoir 版本，严格实现 SpiKL-IP：

```text
每个神经元维护 R_i
每个神经元维护 tau_m_i
根据 SpiKL-IP 规则在线更新 R_i 和 tau_m_i
其余刺激流程、训练轮数、响应窗口和分类器保持一致
```

该版本用于证明 SpiKL-IP 原始形式在本项目任务上的可行性，但它会改变神经元模型，因此不能直接替代当前 Izhikevich 主实验。

### 10.4 实验条件

建议至少设置四组：

```text
1. STDP only
   当前主实验条件。

2. STDP + rate-target IP
   普通目标发放率 IP，只控制平均发放率。

3. STDP + SpiKL-IP-inspired IP
   在 Izhikevich 网络中加入钙浓度估计和指数分布目标。

4. SpiKL-IP-inspired IP only
   关闭 STDP，只保留 SpiKL-IP-inspired IP。
```

如果第 9 个实验已经实现 STP，可额外增加：

```text

```

## 11. 自适应结构进化拓扑实验

### 11.1 实验目的

在当前主实验的随机递归连接网络基础上，引入结构进化机制：不只训练突触权重，还在训练前筛选或优化液体层/BNN 的连接拓扑。

核心问题：

```text
随机拓扑是否已经足够？
经过结构进化筛选的拓扑是否能产生更强的六模式分离性？
结构进化是否能减少沉默神经元和全局爆发？
结构进化与 STDP 是否互补？
```

该实验关注的是网络结构本身的质量，而不是单纯调整 STDP 参数。

### 11.2 实验假设

假设 H1：

```text
经过结构进化筛选的网络，在训练前就具有更好的刺激响应分离性。
```

假设 H2：

```text
Evolved topology + STDP 的最终分类准确率高于 Random topology + STDP。
```

假设 H3：

```text
结构进化可以提高神经元利用率，降低沉默神经元比例，并减少极端同步爆发。
```

### 11.3 基本思想

将递归连接拓扑看作一个可进化对象。

在当前网络中，可以把连接矩阵二值化为染色体：

```text
Chrom[i, j] = 1：神经元 i 到神经元 j 存在突触连接
Chrom[i, j] = 0：神经元 i 到神经元 j 不存在突触连接
```

权重大小仍然按当前 `WeightParams` 初始化或由 STDP 更新，结构进化阶段主要决定“哪些连接存在”。

为了避免改变问题规模，进化时应尽量保持：

```text
神经元数量不变
兴奋/抑制比例不变
总突触数或连接密度近似不变
兴奋性/抑制性突触符号约束不变
不允许自连接
空间距离约束尽量不变
```

这样才能把性能差异归因于拓扑组织方式，而不是简单的突触数量变化。

### 11.4 实验条件

建议设置四组：

```text
1. Random topology + STDP
   当前主实验，作为基线。

2. Evolved topology + no plasticity
   只使用进化拓扑，不启用 STDP，用于评估结构本身的贡献。

3. Evolved topology + STDP
   进化拓扑后执行当前主实验训练流程。

4. Degree-matched random topology + STDP
   保持进化拓扑的入度/出度分布，但随机重连，用于排除度分布本身的影响。
```

如果第 9、10 个实验已经完成，可额外增加：

```text
5. Evolved topology + STDP + STP
6. Evolved topology + STDP + SpiKL-IP-inspired IP
```

用于观察结构进化是否能和稳态机制叠加。

### 11.5 进化阶段流程

结构进化放在完整主训练之前。

推荐流程：

```text
1. 生成 N_pop 个候选网络拓扑。
2. 每个候选拓扑只运行短评估，不做完整 3 轮训练。
3. 对六种刺激各呈现少量 trial，记录网络响应。
4. 计算拓扑适应度 fitness。
5. 选择高 fitness 个体保留。
6. 对保留个体生成后代并变异连接。
7. 重复 G 代。
8. 选出最优拓扑进入完整主实验训练。
```

短评估建议：

```text
预热：5 s
每模式 trial 数：3-5
响应窗口：5-55 ms
STDP：关闭
分类器：不训练或只做快速线性评估
```

进化阶段先关闭 STDP，可以避免适应度同时受到权重学习影响，使结构评价更干净。

### 11.6 染色体与变异方式

**染色体**

```text
Chrom: N × N 二值邻接矩阵
N = 1000
Chrom[i, i] = 0
Chrom[i, j] = 1 表示 i -> j 连接存在
```

如果直接进化 1000×1000 矩阵计算量过大，可先做简化版本：

```text
只进化记录通道附近神经元的局部连接
只进化兴奋性神经元之间的 E -> E 连接
只进化一定比例的可变连接，其余连接固定
```

**变异 1：沉默神经元连接修复**

```text
找到短评估中从未发放的神经元
随机选择一个沉默神经元
为它增加来自活跃神经元的输入连接
或增加它指向局部网络的输出连接
```

该变异用于提高神经元利用率。

**变异 2：弱贡献连接替换**

```text
找到活动相关性长期很低的连接
删除其中一部分
在空间邻近区域新增同等数量连接
```

该变异用于保持连接数不变，同时探索更有效的拓扑。

**变异 3：模块间连接调节**

```text
在 A1/A2 刺激热点相关区域与其他区域之间调整少量长程连接
避免网络只形成局部响应或全局同步响应
```

该变异用于改善六模式刺激的空间传播和分离。

### 11.7 适应度函数

适应度不建议只用最终分类准确率，否则进化阶段会变成昂贵的超参数搜索。

推荐组合适应度：

```text
fitness = separation_score
          + alpha * activity_score
          - beta * burst_penalty
          - gamma * density_penalty
```

其中：

**分离性 separation_score**

可使用状态矩阵秩或类间/类内距离比：

```text
SP = rank(S)
```

或：

```text
separation_score = mean_between_class_distance / mean_within_class_distance
```

其中 `S` 可以是：

```text
所有记录通道的 5-55 ms spike count 矩阵
或所有神经元的二值发放状态矩阵
```

**活动利用率 activity_score**

```text
activity_score = 1 - silent_neuron_ratio
```

用于奖励更多神经元参与响应。

**爆发惩罚 burst_penalty**

```text
burst_penalty = burst-like trial ratio
```

用于惩罚全局同步爆发。

**密度惩罚 density_penalty**

```text
density_penalty = abs(current_synapse_count - target_synapse_count) / target_synapse_count
```

用于防止进化算法简单地通过增加连接数提高响应。

### 11.8 推荐参数

为了控制计算量，先做小规模版本：

```text
N_pop = 20
N_elite = 5
N_offspring_per_elite = 3
G = 10
mutation_rate = 0.5% - 2% 可变连接
short_eval_trials_per_mode = 3
```

如果结果稳定，再扩展：

```text
N_pop = 50
N_elite = 10
N_offspring_per_elite = 5
G = 20-30
short_eval_trials_per_mode = 5
```

每个候选拓扑应使用相同的：

```text
神经元位置
刺激热点
刺激校准参数
初始权重分布规则
短评估刺激序列
```

选出拓扑后，进入当前主实验流程：

## 12. 培养神经元型 LIF 模型替换实验

### 12.1 实验目的

当前主实验使用 Izhikevich 神经元模型，能够产生丰富放电动力学，但它不是直接针对培养神经元网络构建的电流模型。

本实验引入一种更接近培养神经元储层的 LIF 神经元模型，重点加入：

```text
Ca2+ 依赖 K+ 电流
脉冲频率适应
不应期电流
电导型兴奋/抑制突触
Ornstein-Uhlenbeck 噪声电流
```

核心问题：

```text
更接近培养神经元的 LIF 模型是否能复现当前六模式分类结果？
Ca2+ 依赖 K+ 慢适应是否能抑制连续刺激造成的爆发？
电导型突触和 OU 噪声是否改变模式响应的可分性？
Izhikevich 模型和培养神经元型 LIF 模型的结果是否一致？
```

### 12.2 实验假设

假设 H1：

```text
培养神经元型 LIF 模型能在同一六模式刺激任务中达到高于随机基线的分类准确率。
```

假设 H2：

```text
Ca2+ 依赖 K+ 电流会降低连续刺激下的过度发放和 burst-like trial 比例。
```

假设 H3：

```text
如果 LIF 模型的分类结果与 Izhikevich 主实验趋势一致，说明六模式现象不是某一种神经元模型的特例。
```

### 12.3 神经元模型

**兴奋性神经元**

```text
tau_E * dV_E/dt =
    E_L_E - V_E
    + R_in_E * (I_E + I_I + I_KCa + I_ref + I_noise + I_stim)
```

兴奋性神经元包含：

```text
兴奋性突触电流 I_E
抑制性突触电流 I_I
Ca2+ 依赖 K+ 电流 I_KCa
不应期电流 I_ref
OU 噪声电流 I_noise
外部刺激电流 I_stim
```

**抑制性神经元**

```text
tau_I * dV_I/dt =
    E_L_I - V_I
    + R_in_I * (I_E + I_I + I_noise + I_stim)
```

抑制性神经元不加入 `I_KCa` 和 `I_ref`，用于保持模型分工清晰。

### 12.4 Ca2+ 依赖 K+ 电流

兴奋性神经元每次发放后，胞内 Ca2+ 浓度升高：

```text
c(t + dt) = c(t) - c(t) / tau_Ca * dt + c_step * spike(t)
```

K+ 电流为：

```text
I_KCa = g_KCa * c(t) * (E_K - V_E)
```

由于 `E_K` 比膜电位更负，该项通常是负向电流。连续发放越多，`c(t)` 越高，负向 K+ 电流越强，从而产生慢适应和自抑制。

该机制在本实验中的主要作用：

```text
限制连续刺激导致的高频发放
减少全局同步爆发
让刺激响应在多轮训练中更稳定
```

### 12.5 突触与噪声模型

**电导型突触**

当前主实验使用较简化的突触电流累积。LIF 替换实验中建议使用电导型突触：

```text
I_E = g_E * (V_R_E - V)
I_I = g_I * (V_R_I - V)
```

突触电导按指数衰减：

```text
dg_E/dt = -g_E / tau_s_E + gbar_E * pre_spike_E
dg_I/dt = -g_I / tau_s_I + gbar_I * pre_spike_I
```

其中：

```text
V_R_E = 0 mV
V_R_I = -80 mV
tau_s_E = 2 ms
tau_s_I = 5 ms
```

**OU 噪声**

噪声电流使用 Ornstein-Uhlenbeck 过程：

```text
I_noise(t + dt)
  = I_noise(t)
  + (mu - I_noise(t)) / tau_noise * dt
  + sigma * sqrt(dt) * Normal(0, 1)
```

这比独立高斯噪声更平滑，也更接近培养神经元中带时间相关性的背景波动。

### 12.6 实验条件

建议设置六组：

```text
1. Izhikevich + current-based synapse
   当前主实验。

2. Basic LIF
   只替换为基础 LIF，不加入 Ca2+ 适应和不应期电流。

3. LIF + conductance synapse
   使用电导型突触，但不加入 Ca2+ 适应。

4. LIF + conductance synapse + OU noise
   检验带时间相关噪声后响应是否更稳定或更接近真实培养网络。

5. LIF + conductance synapse + Ca2+ K+ current
   检验慢适应对爆发抑制和分类的影响。

6. Full cultured-neuron LIF
   电导型突触 + OU 噪声 + Ca2+ K+ 电流 + 不应期电流。
```

优先实现：

```text
当前 Izhikevich 主实验
Basic LIF
Full cultured-neuron LIF
```

其余作为消融组补充。
