# 主实验解析

本文档解释当前主实验 `run.py` 的完整流程、模型机制、刺激协议、训练设置、输出指标，以及 `src/config.py` 中与实验流程直接相关的参数。训练轮数为 3 轮；每个刺激模式每轮重复 20 个 trial；每轮内刺激顺序固定，不随机打乱。

## 1. 实验目标

主实验模拟一个二维体外神经网络在六种电刺激模式下的响应和学习过程。网络由 Izhikevich 脉冲神经元组成，突触连接具有空间依赖性，兴奋性突触可以通过 STDP 发生权重更新。

核心问题是：经过重复刺激后，网络对不同刺激区域和不同刺激电压的响应是否变得更容易区分。

六种刺激模式来自：

```text
2 个刺激区域 × 3 个刺激电压 = 6 种刺激模式
```

具体为：

```text
A1_100mV
A1_300mV
A1_500mV
A2_100mV
A2_300mV
A2_500mV
```

## 2. 主实验入口

主实验入口是：

```bash
python run.py
```

代码执行顺序在 `run.py` 中组织，主要分为四步：

```text
[1/4] 预热网络
[2/4] 校准刺激
[3/4] 训练
[4/4] 分析与绘图
```

对应的核心类和函数：

```text
src/config.py       参数配置
src/network.py      Izhikevich 网络、突触传播、STDP
src/stimulation.py  热点选择、刺激校准、脉冲电流构建
src/experiment.py   主实验流程编排
src/decoding.py     特征提取与分类
src/visualize.py    结果绘图
```

## 3. 网络初始化

实验开始时，`run.py` 创建：

```python
cfg = Config()
exp = Experiment(cfg)
```

`Experiment` 内部会创建 `ReservoirModel`。网络初始化包括以下内容：

1. 在二维平面上随机放置神经元。
2. 设置兴奋性和抑制性神经元。
3. 随机选择记录通道。
4. 初始化 Izhikevich 神经元参数。
5. 建立距离依赖的突触连接。
6. 初始化兴奋性和抑制性突触权重。
7. 初始化膜电位、恢复变量、突触电流和 STDP trace。

### 3.1 网络规模参数

```python
n_total = 1000
n_exc = 800
n_inh = 200
record_exc = 80
record_inh = 20
```

含义：

```text
n_total     神经元总数
n_exc       兴奋性神经元数量
n_inh       抑制性神经元数量
record_exc  记录的兴奋性神经元数量
record_inh  记录的抑制性神经元数量
```

当前网络共有 1000 个神经元，其中 800 个兴奋性神经元、200 个抑制性神经元。记录通道共 100 个，模拟 MEA 电极记录到的神经元活动。

### 3.2 空间范围参数

```python
plane_width_mm = 3.85
plane_height_mm = 2.10
```

所有神经元被随机均匀放置在一个 `3.85 mm × 2.10 mm` 的二维平面上。这个空间位置会影响：

```text
突触连接概率
刺激热点位置
刺激空间扩散强度
空间活动图
```

### 3.3 连接概率参数

```python
connection_prob = 0.05
distance_lambda_mm = 0.90
```

网络连接概率不是常数，而是随距离衰减：

```text
P(i -> j) = connection_prob × (0.35 + 0.65 × exp(-d / distance_lambda_mm))
```

其中 `d` 是两个神经元之间的空间距离。

参数影响：

```text
connection_prob       控制整体连接密度
distance_lambda_mm    控制连接随距离衰减的速度
```

`distance_lambda_mm` 越大，远距离连接越多；越小，连接越局部化。

### 3.4 神经元动力学参数

模型使用 Izhikevich 神经元：

```text
dv/dt = 0.04v^2 + 5v + 140 - u + I
du/dt = a(bv - u)
if v >= 30: v <- c, u <- u + d
```

代码中兴奋性神经元和抑制性神经元使用不同的参数分布：

```text
兴奋性神经元：Regular Spiking 风格参数
抑制性神经元：Fast Spiking 风格参数
```

这些参数在 `network.py` 内部生成，不在 `config.py` 中单独暴露。

## 4. 背景电流与随机扰动

每个时间步都会计算总输入电流：

```python
I_total = dc_offset + bias_current + syn_current + noise
```

如果当前时间步有外部刺激，还会加上：

```python
I_total += external
```

### 4.1 背景输入参数

```python
dc_offset = 1.5
noise_std = 1.0
synaptic_tau_ms = 5.0
synaptic_current_limit = 50.0
```

含义：

```text
dc_offset        所有神经元共享的基础直流输入
bias_current     每个神经元初始化时生成的一次性静态偏置
noise_std        每个神经元每个时间步的高斯噪声标准差
synaptic_tau_ms  突触电流指数衰减时间常数
synaptic_current_limit  单个神经元突触电流裁剪上限
```

`dc_offset` 控制网络整体兴奋性。调大后神经元更容易自发放电；调小后网络更安静。

`bias_current` 不是 IP 机制。当前代码里它只在网络初始化时随机生成一次：

```python
self.bias_current = rng.normal(0.0, 0.4, size=self.n_total)
```

之后训练过程中不会根据发放率更新它。原来的 IP 机制已经删除；IP 的核心是动态调节 `bias_current` 来逼近目标发放率，而当前版本没有这种动态调节。

`noise_std` 是主实验中的随机扰动来源之一。每个时间步都会采样：

```python
rng.normal(0.0, noise_std, n_total)
```

因此即使刺激顺序不随机，网络内部仍然有背景噪声扰动。若设置为 `0.0`，则去掉这部分运行时随机扰动。

`synaptic_tau_ms` 控制旧突触电流保留多久。当前 `dt_ms = 1.0`，`synaptic_tau_ms = 5.0`，所以每 1 ms 后突触电流乘以：

```text
exp(-1 / 5) ≈ 0.819
```

旧突触电流不会立即归零。如果新 spike 到来，新输入会叠加在旧电流残留上。

`synaptic_current_limit = 50.0` 用于防止删除 STP/IP 后兴奋性反馈过强导致突触电流持续累积。它不是额外可塑性机制，只是数值和动力学稳定保护。

## 5. 突触权重

突触权重参数为：

```python
exc_mu = 0.0
exc_sigma = 0.25
inh_mu = 0.4
inh_sigma = 0.2
w_min = 0.05
w_max = 3.0
```

兴奋性突触初始化为对数正态分布：

```text
w_exc = exp(N(exc_mu, exc_sigma))
```

典型兴奋性权重约为：

```text
exp(0.0) = 1.0
```

抑制性突触初始化为负的对数正态分布：

```text
w_inh = -exp(N(inh_mu, inh_sigma))
```

典型抑制性权重约为：

```text
-exp(0.4) ≈ -1.49
```

`w_min` 和 `w_max` 只用于 STDP 后的兴奋性权重裁剪。当前主实验中，只有兴奋性突触参与 STDP，抑制性突触不会被 STDP 修改。

## 6. STDP 可塑性

当前主实验只保留 STDP，不包含 STP 和 IP。

参数为：

```python
enabled = True
a_plus = 0.004
a_minus = 0.005
tau_pre_ms = 20.0
tau_post_ms = 20.0
```

### 6.1 STDP 的基本含义

STDP 根据 pre 神经元和 post 神经元的发放时间关系调整兴奋性突触权重。

简化理解：

```text
pre 先发放，post 随后发放：突触增强
post 先发放，pre 随后发放：突触减弱
```

代码中维护两个痕迹变量：

```text
pre_trace   记录某个神经元最近是否作为 pre 发放过
post_trace  记录某个神经元最近是否作为 post 发放过
```

每个时间步，trace 会指数衰减：

```python
pre_trace *= exp(-dt / tau_pre_ms)
post_trace *= exp(-dt / tau_post_ms)
```

### 6.2 LTP 参数

```python
a_plus = 0.004
```

当 post 神经元发放时，它接收到的兴奋性输入会根据对应 pre 神经元的 `pre_trace` 增强：

```text
Δw = a_plus × pre_trace
```

`a_plus` 越大，增强越快。

### 6.3 LTD 参数

```python
a_minus = 0.005
```

当 pre 神经元发放时，它指向的兴奋性输出突触会根据目标 post 神经元的 `post_trace` 减弱：

```text
Δw = -a_minus × post_trace
```

当前 `a_minus` 略大于 `a_plus`，用于避免兴奋性权重整体持续膨胀。

### 6.4 时间窗口参数

```python
tau_pre_ms = 20.0
tau_post_ms = 20.0
```

这两个参数控制 spike 痕迹保留多久。数值越大，pre/post spike 间隔较长时仍可能影响权重；数值越小，只有更接近的 spike 时序会改变突触。

## 7. 预热阶段

预热由 `Experiment.warmup()` 执行。

参数为：

```python
dt_ms = 1.0
warmup_ms = 20000.0
warmup_mfr_window_ms = 5000.0
```

含义：

```text
dt_ms                 仿真时间步长
warmup_ms             预热总时长
warmup_mfr_window_ms  用于统计预热活动的最后时间窗口
```

当前设置表示：

```text
预热总时长：20000 ms = 20 s
统计窗口：最后 5000 ms = 最后 5 s
仿真步长：1 ms
预热步数：20000 步
统计窗口步数：5000 步
```

预热阶段没有外部刺激。网络在背景电流、随机噪声、突触传播和 STDP 的作用下自发运行。

预热输出：

```text
warmup_counts  最后 5 秒内 100 个记录通道的 spike count
mfr            记录通道平均发放率
```

预热结束后会保存一次兴奋性权重快照：

```text
post_warmup
```

## 8. 刺激热点选择

热点选择由 `select_hotspots()` 执行。它使用预热最后 5 秒内记录通道的 spike count。

参数为：

```python
top_fraction = 0.10
min_distance_mm = 1.0
dbscan_eps_mm = 0.30
dbscan_min_samples = 3
```

### 8.1 选择高活动通道

记录通道总数为 100，`top_fraction = 0.10` 表示取活动最高的前 10 个通道：

```text
100 × 0.10 = 10
```

这些通道的位置用于寻找刺激热点。

### 8.2 聚类

先使用 DBSCAN 聚类：

```text
eps = 0.30 mm
min_samples = 3
```

如果 DBSCAN 找不到至少两个有效簇，则退回到 KMeans，将高活动通道强制分成两个簇。

每个簇的中心通常按 spike count 加权平均。如果某个簇的 spike count 全为 0，则用普通位置均值作为兜底。

### 8.3 选择两个热点

代码会优先选择总活动量较高、且距离不小于 `min_distance_mm = 1.0` 的两个簇作为 A1 和 A2。

若没有满足最小距离的候选对，则使用前两个簇。

输出：

```text
area1  刺激区域 A1 的中心坐标
area2  刺激区域 A2 的中心坐标
dist   两个热点之间的距离
```

## 9. 刺激校准

刺激校准由 `calibrate_stimuli()` 执行。目标是找到合适的刺激强度缩放 `alpha` 和空间扩散半径 `sigma`，使不同电压下的招募神经元数量接近目标值。

相关参数：

```python
voltages_mv = (100.0, 300.0, 500.0)
target_counts = (20, 80, 200)
alpha_grid = (0.005, 0.010, 0.015, 0.020, 0.030, 0.040, 0.050)
sigma_grid = (0.20, 0.30, 0.40, 0.50, 0.60, 0.70, 0.80, 1.00)
recruitment_threshold = 0.5
```

刺激场公式为：

```text
field(d) = alpha × voltage × exp(-d^2 / (2σ^2))
```

其中：

```text
d        神经元到刺激中心的距离
alpha    刺激场强缩放因子
voltage  刺激电压
sigma    空间扩散半径
```

如果某个神经元满足：

```text
field(d) >= recruitment_threshold
```

则在校准阶段认为它被该刺激电压招募。

目标招募数为：

```text
100 mV -> 约 20 个神经元
300 mV -> 约 80 个神经元
500 mV -> 约 200 个神经元
```

校准会遍历 `alpha_grid × sigma_grid`，选择总误差最小的一组参数。

当前实现中，每个电压最终使用同一个选出的 `sigma`，并记录为：

```text
sigmas[100.0]
sigmas[300.0]
sigmas[500.0]
```

## 10. 刺激电流构建

每个 trial 开始时，根据刺激区域、电压、`alpha`、`sigma` 和脉冲波形构建外部电流。

相关参数：

```python
stimulus_current_gain = 40.0
pulse_phases = ((1, 1.0), (1, -1.0), (3, 0.0))
```

实际注入电流为：

```text
I(d) = alpha × voltage × exp(-d^2 / (2σ^2)) × stimulus_current_gain × phase_scale
```

`pulse_phases` 的每个元素表示：

```text
(持续时间步数, 电流缩放倍数)
```

当前 `dt_ms = 1.0`，所以默认脉冲为：

```text
第 1 ms：正相，scale = +1.0
第 2 ms：负相，scale = -1.0
第 3-5 ms：恢复期，scale = 0.0
```

也就是一个 5 ms 的双相刺激脉冲。

## 11. 训练阶段

训练由 `Experiment.run_rounds()` 执行。

当前训练参数：

```python
trial_duration_ms = 200.0
response_window_start_ms = 5.0
response_window_ms = 55.0
n_rounds = 3
reps_per_mode = 20
```

### 11.1 训练规模

每轮包含 6 种模式，每种模式重复 20 次：

```text
每轮 trial 数 = 6 × 20 = 120
总 trial 数 = 3 × 120 = 360
```

### 11.2 固定刺激顺序

当前版本不随机打乱 trial 顺序。每一轮内刺激顺序固定为：

```text
A1_100mV × 20
A1_300mV × 20
A1_500mV × 20
A2_100mV × 20
A2_300mV × 20
A2_500mV × 20
```

然后进入下一轮，重复同样顺序。

### 11.3 单个 trial 内发生什么

每个 trial 总长 200 ms。流程为：

1. 根据当前模式构建刺激电流。
2. 在 trial 开头施加双相脉冲。
3. 网络继续运行到 200 ms。
4. 每个时间步都进行神经元状态更新、突触电流衰减、spike 传播和 STDP 更新。
5. 只统计刺激开始后 5-55 ms 内记录通道的 spike count 作为响应特征。

也就是说：

```text
0-5 ms       刺激脉冲期，不计入分类特征
5-55 ms      用于分类和发放率统计
55-200 ms    继续参与网络动力学和 STDP，但不计入分类特征
```

每个 trial 会记录：

```text
mode label             当前刺激模式标签
trial_channel_counts   100 个记录通道在响应窗口内的 spike count
trial_mfr              响应窗口内的平均发放率
```

每轮结束后会保存一次兴奋性权重快照：

```text
post_round1
post_round2
post_round3
```

## 12. 功能连接指标

每轮结束后，代码会根据该轮所有 trial 的记录通道响应计算功能连接指标。

输入矩阵为：

```text
round_counts: shape = 120 trials × 100 channels
```

计算过程：

1. 对记录通道之间的活动向量计算相关矩阵。
2. 去掉对角线。
3. 用 `abs(corr) > 0.1` 构建二值图。
4. 使用 NetworkX 和 Louvain 社区检测计算图指标。

输出指标：

```text
avg_correlation    通道间平均相关
global_efficiency  全局效率
modularity         模块化系数
```

这些指标描述刺激响应下记录通道活动的整体相关结构和网络功能组织。

## 13. 解码与分类

训练结束后，`run.py` 会对每一轮分别做分类评估。

### 13.1 原始计数特征

原始特征是每个 trial 的 100 通道 spike count 向量：

```text
feature_raw = [count_ch1, count_ch2, ..., count_ch100]
```

每轮有 120 个样本，每个样本是 100 维。

### 13.2 V100 特征

V100 特征由 `extract_v100()` 生成。

规则：

1. 找出 spike count 大于 0 的通道。
2. 按 spike count 从高到低排序。
3. 取最多前 100 个通道索引。
4. 通道索引用 1-based 编码。
5. 若不足 100 个活动通道，用 0 填充。

V100 不是 spike count 本身，而是活动通道的排序索引特征。

### 13.3 分类器

分类器为：

```text
RandomForestClassifier(n_estimators=100, random_state=42)
```

每轮使用 10 折交叉验证：

```text
KFold(n_splits=10, shuffle=True, random_state=42)
```

注意：训练刺激顺序不随机打乱，但分类评估中的 KFold 默认会 shuffle 样本。这只影响分类评估的数据划分，不影响网络训练过程。

六分类随机基线为：

```text
1 / 6 = 16.7%
```

### 13.4 混淆矩阵

混淆矩阵只对最后一轮计算。当前实现使用最后一轮的原始计数特征做 10 折 out-of-fold 预测：每一折只用训练折拟合 RandomForest，再预测该折未见过的测试样本，最后汇总所有测试折预测结果生成混淆矩阵。

因此，`confusion_matrix.png` 反映的是交叉验证下的泛化混淆情况，不再是训练集内预测的表观结果。

## 16. 需要注意的实验假设

1. 当前训练顺序固定，不再随机打乱。这样更接近“重复刺激一个模式后再切换到下一个模式”的流程。
2. 当前仍然存在背景噪声扰动，因为 `noise_std = 1.0`。
3. 当前只保留 STDP。没有短时程突触可塑性 STP，也没有内在可塑性 IP。
4. STDP 只更新兴奋性突触，抑制性突触权重保持初始化后的固定值。
5. 分类评估中的 KFold 会随机打乱样本划分，但它发生在实验结束后的离线分析阶段，不影响训练刺激顺序。
6. `response_window_start_ms = 5.0` 和 `response_window_ms = 55.0` 决定分类特征统计窗口为 5-55 ms，不代表 trial 在 55 ms 后结束；trial 实际持续 200 ms。

## 17. 一句话流程总结

主实验先构建一个 1000 神经元的空间化 Izhikevich 网络，预热 20 秒后从高活动记录通道中选出两个刺激热点，自动校准三种电压的空间刺激范围，然后按固定顺序对六种模式进行 3 轮重复刺激；训练过程中兴奋性突触通过 STDP 更新，最后使用记录通道在刺激后 5-55 ms 内的 spike count 对六种模式进行分类评估，并输出学习曲线、发放率、权重分布、空间响应和功能连接指标。
